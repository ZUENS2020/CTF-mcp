# CTF 经验写作指南

本指南帮助您有效地记录和整理 CTF 挑战中积累的经验。

---

## 目的

经验池 (LESSONS.md) 的目标是:

1. **避免重复踩坑** - 记录常见错误和解决方案
2. **积累技术** - 记录成功的技术和方法
3. **快速参考** - 比赛中快速查找类似问题的解法
4. **知识传承** - 与队友分享经验

---

## 写作原则

### 1. 及时性

- 挑战结束后立即记录
- 不要依赖记忆,现场记录

### 2. 简洁性

- 核心信息,不要废话
- 使用列表和代码块
- 避免冗长的解释

### 3. 可操作性

- 记录具体命令和代码
- 说明什么情况下使用
- 给出已知的问题和限制

### 4. 分类清晰

- 按类别(CTF 领域)组织
- 使用一致的标签
- 便于快速检索

---

## 格式模板

### 基本结构

```markdown
## [类别] 经验标题

- **类别**: 英文缩写 (中文全称)
- **信号/触发点**: 在什么情况下会用到这个技术
- **技术/方法**: 具体使用的技术
- **踩坑记录**: 遇到的问题和解决方案
- **相关题目**: 相关的 CTF 题目
- **参考资料**: 相关的文档或链接
- **记录时间**: YYYY-MM-DD HH:MM:SS

---
```

### 示例

```markdown
## [Pwn]格式化字符串泄露libc地址

- **类别**: Pwn (二进制漏洞利用)
- **信号/触发点**: 程序使用 printf(s) 或类似函数,且没有手动设置缓冲区
- **技术/方法**: 使用 %p 或 %x 泄露栈上的地址,配合 puts 或 write 泄露 got 表
- **踩坑记录**: 需要注意格式化字符串的偏移量,可通过 padding 确定
- **相关题目**: 2024-XCTF-babyheap, RealWorld-CTF-calc
- **参考资料**: https://github.com/Gallopsled/pwntools-tutorial
- **记录时间**: 2024-03-15 18:30:00

---
```

## 常见错误

### ❌ 错误示例

```markdown
# 这个技术很厉害
今天做了一道题,发现了一个技术,感觉很好用,记录一下。
```

**问题**: 缺乏具体信息,无法复用

### ✅ 正确示例

```markdown
## [Pwn] House of Spirit 快速分配

- **信号/触发点**: 需要在任意地址分配堆块,且附近有可控的 fake chunk
- **技术/方法**: 在目标地址构造 fake chunk,调用 free() 触发 malloc 返回该地址
- **踩坑记录**: 需要注意 next chunk 的 size 要对齐,否则会被 unlink 检查拦截
- **相关题目**: 2024-HITCON-sleepy
```

## 进阶技巧

### 链接相关条目

如果新经验与已有条目相关,引用它:

```markdown
## [Pwn] Fastbin Dup 进阶

- **相关条目**: 参见 [Pwn] Fastbin Dup 基础
```

### 添加变体

一个技术有多种变体时,使用子标题:

```markdown
## [Pwn] Fastbin Dup

### 变体1: Fastbin Dup into Stack

### 变体2: Fastbin Dup to Control PC
```

### 记录失败案例

同样重要:

```markdown
## [Web] SSTI Jinja2 RCE 失败案例

- **信号/触发点**: 发现 {{ }} 模板注入
- **技术/方法**: 尝试 {{ config }} 或 {{ ''.__class__ }}
- **踩坑记录**: 过滤了 `class`, `_`, `__`, 无法构造属性链
- **替代方案**: 寻找其他注入点或利用 SSTI 到 LFI
```
